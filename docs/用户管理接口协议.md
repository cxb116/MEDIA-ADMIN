# 用户管理 CRUD 接口协议

## 基础信息

- **Base URL**: `/api/user`
- **认证方式**: Bearer Token (Header: `Authorization: Bearer <token>`)
- **响应格式**: JSON

---

## 1. 查询用户列表

**请求**

```http
GET /api/user/list
```

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | number | 是 | 当前页码 |
| size | number | 是 | 每页条数 |
| userName | string | 否 | 用户名（模糊搜索） |
| userPhone | string | 否 | 手机号 |
| userEmail | string | 否 | 邮箱 |
| status | string | 否 | 状态：1-在线 2-离线 3-异常 4-注销 |

**响应**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [
      {
        "id": 1,
        "avatar": "https://...",
        "status": "1",
        "userName": "张三",
        "nickName": "昵称",
        "userGender": "男",
        "userPhone": "13800138000",
        "userEmail": "test@example.com",
        "userRoles": ["admin", "user"],
        "createBy": "admin",
        "createTime": "2024-01-01 12:00:00",
        "updateBy": "admin",
        "updateTime": "2024-01-01 12:00:00"
      }
    ],
    "current": 1,
    "size": 20,
    "total": 100
  }
}
```

**响应字段说明**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 用户ID |
| avatar | string | 头像URL |
| status | string | 状态：1-在线 2-离线 3-异常 4-注销 |
| userName | string | 用户名 |
| nickName | string | 昵称 |
| userGender | string | 性别：男/女 |
| userPhone | string | 手机号 |
| userEmail | string | 邮箱 |
| userRoles | string[] | 角色编码数组 |
| createBy | string | 创建人 |
| createTime | string | 创建时间 |
| updateBy | string | 更新人 |
| updateTime | string | 更新时间 |

## 2. 新增用户

**请求**

```http
POST /api/user/create
Content-Type: application/json
```

**请求体**

```json
{
  "userName": "张三",
  "userPhone": "13800138000",
  "userGender": "男",
  "userRoles": ["admin"]
}
```

**请求参数说明**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| userName | string | 是 | 用户名 (2-20字符) |
| userPhone | string | 是 | 手机号 (11位，格式：1[3-9]xxxxxxxxx) |
| userGender | string | 是 | 性别：男/女 |
| userRoles | string[] | 是 | 角色编码数组 |

**响应**

```json
{
  "code": 200,
  "message": "添加成功",
  "data": {
    "id": 101,
    "userName": "张三"
  }
}
```

## 3. 更新用户

**请求**

```http
POST /api/user/update
Content-Type: application/json
```

**请求体**

```json
{
  "id": 101,
  "userName": "张三",
  "userPhone": "13800138000",
  "userGender": "男",
  "userRoles": ["admin", "editor"]
}
```

**请求参数说明**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 用户ID |
| userName | string | 否 | 用户名 (2-20字符) |
| userPhone | string | 否 | 手机号 (11位) |
| userGender | string | 否 | 性别：男/女 |
| userRoles | string[] | 否 | 角色编码数组 |

**响应**

```json
{
  "code": 200,
  "message": "更新成功",
  "data": null
}
```

---

## 4. 删除用户

**请求**

```http
POST /api/user/delete/{id}
```

**路径参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | number | 是 | 用户ID |

**响应**

```json
{
  "code": 200,
  "message": "删除成功",
  "data": null
}
```

---

## 5. 批量删除用户

**请求**

```http
POST /api/user/batchDelete
Content-Type: application/json
```

**请求体**

```json
[1, 2, 3]
```

**响应**

```json
{
  "code": 200,
  "message": "批量删除成功",
  "data": null
}
```

## 统一响应结构

```typescript
interface ApiResponse<T> {
  code: number      // 200-成功, 其他-失败
  message: string   // 响应消息
  data: T           // 响应数据
}
```

---

## 分页响应结构

```typescript
interface PaginatedResponse<T> {
  records: T[]      // 数据列表
  current: number   // 当前页码
  size: number      // 每页条数
  total: number     // 总条数
}
```

| code | 说明 |
|------|------|
